========================================================
  动物庄园查卡站 · 内容配置说明（傻瓜式）
========================================================

这个 config 文件夹 = 整个网站的「内容总开关」。
你只改这里的文件，然后在上一层目录运行一次：

    python build.py

网站的卡牌、规则书、FAQ、关键词、标题等就会全部自动更新。
（改完记得刷新浏览器；导出/预览要用 start.bat 起本地服务器。）

--------------------------------------------------------
各文件管什么
--------------------------------------------------------

1) cards_characters.csv  —— 角色牌（主卡角色 + 副卡角色）
2) cards_functions.csv   —— 功能牌（副卡功能）
   · 用 Excel / WPS / 记事本都能打开。
   · 第一行是表头，不要动；从第二行起一行 = 一张卡。
   · 新增卡：在最后加一行，按表头逐列填写即可。
   · 「编号」列决定卡图：编号要和 ../build.py 里 IMAGE_MAP 的对应关系一致；
     没有卡图的卡会显示「暂无卡图」占位，不影响其它功能。

3) rules.md  —— 规则书正文
   · 纯文本，用「##／###／####」表示大/中/小标题（井号越多级别越小）。
   · 「- 」开头是一条列表项；前面缩进 2 个空格表示子项。
   · 「> 」开头是一段提示框（高亮注释）。
   · **两个星号**包起来 = 加粗。
   · {关键词} 会自动高亮并显示释义。
   · {{glossary}} 这一行会被自动替换成“关键词速查表”，别删。
   · 左侧目录会根据标题自动生成，无需手动维护。

4) faq.md  —— 常见问题
   · 「@ 分类名」给问题分组，FAQ 页顶部会出现分类筛选标签。
   · 每个「## 」开头是一个问题，紧跟其后的行是答案。
   · 答案里同样支持 **加粗** 和 {关键词}。
   · 想让某条问答出现在某些卡的图鉴详情里，在答案前加一行：
        卡牌：卡名1，卡名2
     （卡名要和卡表一致；点开这些卡时，详情里会显示这条问答）

5) keywords.csv  —— 关键词释义（两列：关键词,释义）
   · 新增关键词：加一行；卡牌/规则里写 {新词} 就会自动高亮显示释义。

6) site.txt  —— 站点设置（标题、副标题、反馈邮箱、页脚）
   · 每行「键 = 值」，改右边的值即可。
   · feedback_email 改了之后，玩家反馈会发到新邮箱（需重新激活 FormSubmit）。

--------------------------------------------------------
最常见的两件事
--------------------------------------------------------
· 加一张卡 → 编辑 cards_*.csv 加一行 → 跑 build.py
· 改规则书文字 → 编辑 rules.md → 跑 build.py
